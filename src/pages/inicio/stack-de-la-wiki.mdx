Esta wiki está pensada como un sistema simple de mantener: contenido en archivos, navegación clara y UI suficiente para que leer no se sienta pesado.

## Capas del stack

| Capa | Rol en la wiki |
| --- | --- |
| **Astro** | Da estructura al sitio, define rutas y mantiene el renderizado ligero. |
| **MDX / Markdown** | Permite escribir contenido con formato rico sin salir del flujo editorial. |
| **React** | Se usa en los bloques interactivos y componentes reutilizables de la home. |
| **Tailwind CSS** | Mantiene consistencia visual sin duplicar estilos en cada vista. |
| **Layout system** | Une la navegación, el contenedor de lectura y el pie de página en una experiencia coherente. |
| **Sidebar dinámica** | Organiza la navegación por dominios y crece de forma automática a medida que agrego nuevos documentos. |
| **Tipografía y tokens** | Le dan identidad visual y hacen que el texto técnico se lea con menos fricción. |

## Por qué este stack

- Prioriza velocidad de escritura para que crear contenido sea más importante que pelear con la herramienta.
- Mantiene el sitio simple de desplegar y fácil de extender.
- Se adapta bien a una wiki que crecerá por secciones, no por páginas aisladas.
- Separa bien la parte editorial de la parte visual, que es justo lo que este tipo de proyecto necesita.
